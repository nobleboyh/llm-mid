Why the graph helps agents
  • Agents trace class relationships through the structured graph — no need to grep raw source files
  • Cleaner, pre-organised context means the model spends tokens on reasoning, not parsing
  • VS Code extension surfaces graph context directly to coding agents in-editor
Faithfulness — what it means
Faithfulness measures whether the AI answer is grounded in the provided context. A score of 1.0 means every claim in the answer can be traced back to the source material. With the knowledge graph, agents get structured, accurate context — so their answers stick to the facts.
Key result
When the AI agent explored the codebase using the knowledge graph, Faithfulness jumped from 0.929 to 1.000 — a perfect score. The graph isn't just a UX win — it produces verifiably grounded answers.

GateMid — Feature 2
9 / 18
Feature 2 — Token Compression
60–95% fewer tokens. Same answer quality.
Headroom ASGI middleware compresses context before it reaches the LLM — zero intelligence loss.
Tokens sent — before vs after compression
Three compression strategies
SmartCrusher
Compresses JSON, FHIR, and HL7 payloads — purpose-built for the verbose structured data common in healthcare codebases.
CodeCompressor
AST-based compression for source code — strips boilerplate while preserving semantic meaning.
CacheAligner
Aligns prompts to KV cache prefixes — reduces redundant computation on repeated context patterns.
"Compress tool outputs, logs, files, and RAG chunks before they reach the LLM. 60–95% fewer tokens, same answers." — Headroom, 16k+ stars on GitHub
